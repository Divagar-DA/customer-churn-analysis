# Customer ChurnAnalysis (SQL)

## Overview
SQL project analyzing customer usage data to identify churn risk and engagement levels using real-world business logic.

## Key Highlights
- Calculated latest activity and inactivity duration per customer
- Identified high churn risk users using date-based analysis
- Segmented customers into High Churn Risk, Low Engagement, and Highly Engaged
- Used window functions to retrieve latest customer activity records
- Produced a final business-ready customer summary table

## Tools & Concepts
SQL Server | Aggregations | CASE | CTEs | Window Functions
